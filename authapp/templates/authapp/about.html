L'analyse métier assistée par IA

Le Projet Engineeria consiste à porter sur l'intranet Naldeo une interface web conviviale, orientée métier, pour permettre l'analyse de documents assistée par intelligence artificielle.

Les fonctionnalités de base de la version bêta de démonstration de Naldéa permettent actuellement :

  • Un login / gestion des utilisateurs
  • La création/suppression de dossiers d'analyse (groupe de documents)
  • Le téléchargement et la vectorisation des fichiers à analyser en mode asynchrone.
  • De conduire (et sauvegarder) des discussions avec les documents mémorisés
  • De lancer des séquences d'analyse standardisés (sommaire, résumé, couts d'analyse, affichage géolocalisé du contenu des documents)
  • De lancer des prompts métiers (webscrapping, extraction de caractéristiques techniques, génération de rapport Excel, vérification de cohérence inter-documentaire)

Techno version béta

La version béta de démonstration de l'interface utilisateur Naldéa est basée sur les technologies :

  • Django / SQLite pour le serveur web et la gestion des droits d'authentification
  • HTML5 / Bootstrap5/ jQuery pour le front 'single page application'l'API python OPENAI pour la vectorisation et le questionnement des documents, rassemblée sous un ensemble de classes informatiques, dénommé Engineeria.
  • Git pour le versionning (dépôt sur gitlab)

Perspectives

Le projet en version bêta doit permettre de :

  • Valider les temps de réponse de l'api OpenAI pour plusieurs utilisateurs connectés simultanément (en mémorisation et questionnement)
  • Tester notamment la stabilité et la scalabilité de la solution OpenAI pour un déploiement en version 1
  • Recueillir l'intérêt de ce type d'outils auprès des collaborateurs "métiers" (maîtrise d'œuvre classique construction et assistance à maîtrise d'ouvrage)
  • Valider l'emploi des technologies mises en œuvre dans la version démonstration pour un développement interne ou suggérer d'autres solutions, le recours à d'autres librairies actuellement en pleine apparition, en fonction des compétences actuelles (Angular vs Jquery, Naldia vs langchain, openai vs Bard vs Llama, sql vs chroma, guidance, etc)Tester la possibilité d'intégrer le sharepoint Microsoft au système d'analyse documentaire
